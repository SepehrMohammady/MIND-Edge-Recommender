FP32 → INT8 → Binary: مقایسه دقت، اندازه و انرژی

۵.۱ چرا Quantization؟

Quantization وزن‌های float32 مدل را به نوع داده کوچکتر تبدیل می‌کند. نتیجه: مدل کوچکتر، استنتاج سریع‌تر، مصرف انرژی کمتر.

۵.۲ سه سطح Precision

FP32 — دقت کامل

وزن‌ها: [-0.423, 1.872, -0.091, ...] هر عدد: 32 bit = 4 byte مزیت: دقت بالا، محاسبه ساده مشکل: حافظه زیاد، انرژی بالا

INT8 — کوانتیزه شده

وزن‌ها: [-54, 127, -12, ...] هر عدد: 8 bit = 1 byte فرمول: w_int8 = round(w_fp32 / scale) مزیت: 4× کوچکتر، سخت‌افزار بهینه‌شده (ARM Cortex-M) مشکل: کمی افت دقت (اما QAT آن را جبران می‌کند)
💡 INT8 ≈ FP32
در پروژه ما INT8 تقریباً همان دقت FP32 را دارد (گاهی کمی بالاتر، به‌خاطر اثر regularization در QAT). این تفاوت‌ها کوچک‌اند و در حد نوسانِ یک seed — پس ادعا نمی‌کنیم INT8 واقعاً از FP32 بهتر است. نکتهٔ مهم: کوانتیزه‌کردنِ این مدلِ کوچک دقت را تقریباً تغییر نمی‌دهد ولی اندازه و انرژی را کم می‌کند.

Binary — یک‌بیتی

وزن‌ها: [-1, +1, -1, +1, -1, ...] ← فقط دو مقدار! XNOR-Net: لایه اول/آخر و Byte Embedding = FP32 لایه‌های میانی = Binary (sign × scale) مزیت: کوچکترین اندازه ممکن مشکل: STM32H7 POPCOUNT ندارد → سرعت واقعی کمتر از INT8

۵.۳ PTQ vs QAT

روشتوضیحمزیتمشکل
PTQ
Post-Training Quantization		مدل آموزش‌دیده FP32 را مستقیماً کوانتیزه کنسریع، بدون آموزش مجددکمی افت دقت
QAT
Quantization-Aware Training		در حین آموزش، quantization شبیه‌سازی کندقت بهتر، گاهی بهتر از FP32نیاز به آموزش مجدد

۵.۴ کد Quantization

Python — src/quantize.py (ساده‌شده)
from src.quantize import convert_to_quant

# یک مدل آموزش‌دیدهٔ FP32 را به INT8 یا Binary تبدیل کن. لایه‌های داخلی
# Conv1d/Linear با fake-quant + STE کوانتیزه می‌شوند؛ embedding و
# proj_in/head به‌صورت FP32 می‌مانند (روش XNOR-Net).
q_int8   = convert_to_quant(model, "int8")     # PTQ؛ برای QAT دوباره train کن
q_binary = convert_to_quant(model, "binary")

# هستهٔ کوانتیزه‌سازی در src/quantize.py (ساده‌شده):
def fake_quant_weight(w, bits):
    if bits == 1:                          # Binary: sign × میانگین|w| (per-channel)
        scale = w.abs().mean(dim=1, keepdim=True)
        return _SignSTE.apply(w) * scale
    qmax  = 2 ** (bits - 1) - 1        # INT8: qmax = 127
    scale = w.abs().amax(dim=1, keepdim=True) / qmax
    return _RoundSTE.apply((w / scale).clamp(-qmax-1, qmax)) * scale
✅ بازیابی دقت Binary (هدف اصلی ما)
Binaryِ ساده ضعیف است (Micro-NAS/Binary ≈ ۰٫۵۲۱، نزدیکِ تصادفی). اما با تکنیک‌های ReActNet (RSign/RPReLU) + میان‌بر Bi-Real و مقداردهیِ اولیه با distillation، دقتِ binary به ۰٫۵۷۲ در ۱۸۶KB می‌رسد — فقط ۰٫۰۳۵ AUC پایین‌تر از baselineِ کامل، با دو مرتبه‌بزرگی حافظهٔ کمتر. Binary گزینهٔ کم‌ترین‌فوت‌پرینت است؛ INT8 هنوز در دقت جلوتر.

۵.۵ XNOR-Net — لایه‌های Full-Precision

⚠️ نکته مهم
در پروژه ما (مثل XNOR-Net) لایه‌های زیر همیشه FP32 نگه داشته می‌شوند:
  • Byte Embedding: جدول 257 × 64 (256 بایت + pad)
  • اولین Conv: ورودی binary نباشد
  • Linear head: خروجی با دقت کامل
فقط لایه‌های میانی conv و linear کوانتیزه می‌شوند.

تمرین ۵.۲ — اجرای مدل ONNX

Python
import onnxruntime as ort, numpy as np, time

sess = ort.InferenceSession("artifacts/content_encoder.onnx",
       providers=["CPUExecutionProvider"])

x = np.zeros((1, 128), dtype=np.int64)   # batch=1, len=max_title_bytes=128

# اجرا و اندازه‌گیری زمان
t0 = time.perf_counter()
for _ in range(100):
    emb = sess.run(None, {"title_bytes": x})[0]
ms = (time.perf_counter() - t0) / 100 * 1000

print(f"Embedding shape: {emb.shape}")
print(f"Latency CPU: {ms:.2f} ms/inference")
