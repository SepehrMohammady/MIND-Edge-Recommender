Knowledge Distillation، teacher چندزبانه و student بایت‌محور

۳.۱ Knowledge Distillation چیست؟

Knowledge Distillation (تقطیر دانش) روشی است که در آن یک مدل سبک (Student) یاد می‌گیرد رفتار مدل بزرگ‌تر (Teacher) را تقلید کند. نتیجه: دقت نزدیک به teacher، اندازه بسیار کوچکتر.

Teacher (بزرگ، دقیق، کُند) Student (کوچک، سریع) ┌──────────────────────┐ ┌────────────────┐ │ paraphrase-multilingual│ → anchor → │ Byte-level CNN │ │ MiniLM-L12-v2 │ embeddings │ │ │ 117 MB, 384-dim │ │ 204 KB, 384-dim│ └──────────────────────┘ └────────────────┘ (یخ‌زده، آموزش نمی‌بیند) ↑ Matryoshka Cosine Loss (student باید embedding teacher را تقلید کند)

۳.۲ Teacher — مدل چندزبانه

ما از paraphrase-multilingual-MiniLM-L12-v2 به عنوان teacher استفاده می‌کنیم. این مدل قبلاً با میلیون‌ها جمله در زبان‌های مختلف آموزش دیده.

Anchor Embeddings

Teacher فقط روی عناوین انگلیسی MIND اجرا می‌شود و یک anchor embedding برای هر مقاله می‌سازد. سپس student یاد می‌گیرد ترجمه‌های هر زبان را به همان anchor نزدیک کند.

✅ ایده هوشمندانه
چون xMIND ترجمه‌های موازی دارد (همه زبان‌ها همان مقاله را توصیف می‌کنند)، هر ترجمه باید به anchor یکسانی نزدیک شود. Student یاد می‌گیرد معنا را بشناسد، نه زبان را!

۳.۳ Student — Byte-level Char-CNN

Student یک شبکه عصبی کانولوشنی بایت‌محور است. به جای یادگیری روی کلمات، روی بایت‌های UTF-8 عمل می‌کند.

ورودی: "Sports News" ↓ encode UTF-8 [83, 112, 111, 114, 116, 115, ...] ← byte ids (0-255) ↓ Byte Embedding Table (257 × 64) ← 256 بایت + 1 pad [vec₀, vec₁, vec₂, ...] ← embedding هر بایت ↓ Depthwise-Separable 1-D Conv (چند لایه) [feature_maps...] ↓ Masked Mean Pooling [384-dim vector] ← embedding مقاله ↓ Linear head [out_dim vector] ← خروجی نهایی

۳.۴ Matryoshka Loss

Matryoshka Representation Learning می‌گوید: ابتدای embedding باید به تنهایی هم مفید باشد. اگر embedding 384-بعدی داریم، prefix 64-بعدی آن هم باید کار کند.

💡 چرا مهم است؟
روی دستگاه‌های بسیار ضعیف می‌توانیم از 64 یا 128 بعد اول استفاده کنیم و از ذخیره‌سازی کمتری بهره‌مند شویم — بدون آموزش مجدد.

۳.۵ کد آموزش Student

Python — src/student.py (ساده‌شده)
import torch, torch.nn as nn

class ByteLevelEncoder(nn.Module):
    def __init__(self, channels=64, depth=5, out_dim=384):
        super().__init__()
        self.embed = nn.Embedding(257, 64)   # 256 بایت + pad؛ سپس proj→channels (src/student.py)
        self.convs = nn.Sequential(*[
            DepthwiseSepConv(channels) for _ in range(depth)
        ])
        self.head = nn.Linear(channels, out_dim)

    def forward(self, byte_ids):             # (B, L)
        mask = (byte_ids > 0).float()
        x = self.embed(byte_ids)             # (B, L, C)
        x = self.convs(x.transpose(1,2)).transpose(1,2)
        x = (x * mask.unsqueeze(-1)).sum(1) / mask.sum(1, keepdim=True)
        return self.head(x)                   # (B, out_dim)

# تابع Matryoshka Loss
def matryoshka_loss(pred, target, dims=[64, 128, 256, 384]):
    total = 0
    for d in dims:
        total += 1 - nn.functional.cosine_similarity(
            pred[:, :d], target[:, :d]).mean()
    return total / len(dims)

۳.۶ واژه‌نامه اصطلاحات کلیدی

هر اصطلاحی که در این فصل دیدی، ساده و با مثالِ همین پروژه:

اصطلاحیعنی چه؟
embedding
(بردار نهفته)		تبدیل یک چیز گسسته (یک بایت، یک کلمه) به برداری از اعداد که شبکه می‌تواند رویش حساب کند. مثل یک جدول جست‌وجو: id → بردار.
Byte Embedding
(امبدینگ بایت)		جدول ما برای ۲۵۶ مقدار ممکنِ بایت UTF-8 (+۱ برای pad). هر بایت → بردار ۶۴‌تایی (byte_embed_dim=64). یک جدول برای هر ۱۴ زبان → هیچ واژگانِ جداگانه‌ای برای هر زبان لازم نیست.
padding
(پُرکردن)		عنوان‌ها طول متفاوت دارند ولی تنسور باید اندازهٔ ثابت داشته باشد. جاهای خالی را با id صفر پر می‌کنیم تا به max_title_bytes=128 برسد. pad معنا ندارد و باید نادیده گرفته شود.
Conv Blocks
(بلوک‌های کانولوشن)		فیلترهای کانولوشن ۱بُعدی که روی دنبالهٔ بایت‌ها سُر می‌خورند و الگوهای محلی را یاد می‌گیرند (n-gramهای بایتی ≈ زیرکلمه/تکواژ). ما نوع depthwise-separable را به‌کار می‌بریم که سبک و مناسب MCU است.
Masked Mean Pool
(میانگین‌گیری ماسک‌دار)		دنبالهٔ ۱۲۸ بردار → یک بردار، با میانگین‌گیری روی موقعیت‌ها. «ماسک‌دار» یعنی موقعیت‌های pad در میانگین حساب نمی‌شوند تا پُرکننده‌ها نتیجه را خراب نکنند.
Linear Head
(سرِ خطی)		آخرین لایه، یک nn.Linear(channels → out_dim). بردار جمع‌شده را به اندازهٔ خروجی نهایی تبدیل می‌کند. «head» یعنی لایهٔ پایانی که نتیجه را می‌سازد.
out_dim
(بُعد خروجی)		اندازهٔ بردار خبرِ تولیدشده (مثلاً ۳۸۴). باید با بُعد teacher برابر باشد تا student بتواند آن را تقلید کند. NAS همین مقدار را جست‌وجو می‌کند.
anchor / anchor embedding
(لنگر)		بردارِ هدفی که student کپی می‌کند. teacherِ یخ‌زده عنوانِ انگلیسی هر مقاله را کد می‌کند → بردار ۳۸۴‌تایی = «لنگر». چون ترجمه‌ها موازی‌اند، هر زبان به همان لنگر نزدیک می‌شود → یک فضای معنایی مشترک.
Matryoshka
Representation Learning		طوری آموزش بده که بردار حتی با نگه‌داشتن فقط چند بُعدِ اول هم مفید باشد — مثل عروسک‌های تودرتوی روسی (ماتریوشکا). loss، پیشوندها را در ۶۴/۱۲۸/۲۵۶/۳۸۴ بُعد مقایسه می‌کند.
prefix
(پیشوند)		چند عددِ اولِ یک بردار. vec[:64] = پیشوندِ ۶۴‌تایی. MRL باعث می‌شود همین پیشوندِ کوتاه هم به‌تنهایی معنادار باشد (برای دستگاه‌های کم‌حافظه).
🔗 جریان کامل
بایت‌های عنوان → Byte Embedding → Conv Blocks → Masked Mean Pool → Linear Head (به اندازهٔ out_dim) → embeddingِ student؛ که با loss به anchorِ teacher نزدیک می‌شود، و Matryoshka تضمین می‌کند هر prefix هم کار کند.

۴ چرا teacher فقط روی عناوین انگلیسی اجرا می‌شود (نه ترجمه‌ها)؟

چون xMIND ترجمه موازی دارد — همه زبان‌ها همان مقاله را توصیف می‌کنند. Anchor انگلیسی نمایانگر «معنا» است. Student یاد می‌گیرد هر ترجمه را به همان معنا نزدیک کند. این روش Multilingual Knowledge Distillation (Reimers & Gurevych, 2020) است.

تمرین ۳.۲ — اجرای student روی یک عنوان

Python
import torch
from src.config import load_config
from src.student import ByteCNNEncoder

cfg = load_config()
s = cfg["student"]
# student.pt یک state_dict است؛ اول مدل را بساز، بعد وزن‌ها را بارگذاری کن
model = ByteCNNEncoder(s["byte_embed_dim"], s["channels"], s["depth"], s["out_dim"])
model.load_state_dict(torch.load("artifacts/student.pt", map_location="cpu"))
model.eval()

title = "Sports news from around the world"
L = cfg["data"]["max_title_bytes"]
byte_ids = [b + 1 for b in title.encode("utf-8")][:L]   # +1 چون 0 برای pad رزرو است
byte_ids += [0] * (L - len(byte_ids))

with torch.no_grad():
    emb = model(torch.tensor([byte_ids]))
print(f"Embedding shape: {emb.shape}")   # (1, out_dim)
print(f"Norm: {emb.norm().item():.4f}")
