۶.۱ مسئله رتبه‌بندی در Impression

در هر impression، لیستی از مقالات نمایش داده شده و برخی کلیک شده‌اند. وظیفه مدل: مقالاتی که کلیک می‌شوند را بالاتر رتبه‌بندی کند.

Impression: تاریخچه کاربر: [N1, N2, N5] (قبلاً کلیک شده) کاندیداها: [N10, N11, N12, N13, N14] برچسب‌ها: [0, 1, 0, 1, 0 ] ↑کلیک نشد ↑کلیک شد مدل باید N11 و N13 را بالاتر بیاورد.

۶.۲ معیارهای ارزیابی

AUC — Area Under the ROC Curve

احتمال اینکه مدل یک مقاله کلیک‌شده را بالاتر از یک مقاله نکلیک‌شده رتبه‌بندی کند.

📌 تفسیر AUC
AUC=0.5 → تصادفی | AUC=0.6 → خوب | AUC=0.65 → عالی | AUC=1.0 → کامل

MRR — Mean Reciprocal Rank

میانگین معکوس رتبه اولین مقاله کلیک‌شده. اگر اولین کلیک در رتبه ۱ باشد: MRR=1. اگر رتبه ۵ باشد: MRR=0.2.

مثال: کاندیداها: [A, B, C, D, E] ← رتبه‌بندی مدل برچسب‌ها: [0, 0, 1, 0, 1] اولین کلیک در رتبه 3 → MRR = 1/3 ≈ 0.333

nDCG — Normalized Discounted Cumulative Gain

علاوه بر رتبه، تعداد کلیک‌های درست را هم در نظر می‌گیرد. nDCG@5 و nDCG@10 نسخه‌های رایج هستند.

DCG@5 = Σ (برچسب_i / log2(رتبه_i + 1)) برای i=1 تا 5 nDCG@5 = DCG@5 / IDCG@5 (مقدار بهینه)
معیارچه می‌سنجدبازه
AUCتوانایی کلی تمایز کلیک/غیرکلیک0 – 1
MRRرتبه اولین کلیک0 – 1
nDCG@5کیفیت top-5 نتایج0 – 1
nDCG@10کیفیت top-10 نتایج0 – 1

۶.۵ کد ارزیابی

Python — src/metrics.py
from src import metrics
import json

# results_summary.json کلیدهای best_arch / baseline / matrix / languages دارد
results = json.load(open("artifacts/results_summary.json"))
for row in results["matrix"]:        # matrix یک لیست از ردیف‌هاست
    print(f"{row['arm']}/{row['precision']}: AUC={row['auc']:.3f}, MRR={row['mrr']:.3f}")

# محاسبهٔ معیارها برای یک impression با metrics.evaluate
imp = {"labels": [0, 1, 0, 1, 0],
       "scores": [0.2, 0.8, 0.3, 0.7, 0.1]}
m = metrics.evaluate([imp])     # میانگین روی همهٔ impressionها
print(f"AUC: {m['auc']:.4f}, MRR: {m['mrr']:.4f}, nDCG@10: {m['ndcg@10']:.4f}")

۶.۵ شروع سرد (Cold-Start)

❄️ آیا prior برای کاربر جدید کار می‌کند؟
برای کاربرِ جدیدِ بدون تاریخچه، مدلِ مبتنی‌بر‌تاریخچه در سطح تصادفی است (AUC = ۰٫۵۰۰ — هیچ سیگنالی ندارد). اما رتبه‌بندیِ کاندیداها بر اساس prior جمعیتیِ موضوع (وزنِ دسته‌بندی‌ها که offline صادر می‌شود) کاربرِ سرد را به ۰٫۵۵۱ می‌رساند. یعنی prior صادرشده واقعاً برای کاربران جدید ارزش دارد — پاسخِ مستقیم به مسئلهٔ cold-start.
